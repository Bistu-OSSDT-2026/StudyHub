# 贡献指南

感谢你对 StudyHub 自习室预约系统的关注！

## 如何贡献

### 报告 Bug
如果你发现了 Bug，请通过 GitHub Issues 提交，并包含以下信息：
- 清晰的标题
- 复现步骤
- 期望行为与实际行为
- 环境信息（操作系统、Python 版本、Node.js 版本）

### 提交功能建议
欢迎通过 Issues 提出新功能建议，提交前请先检查是否已有类似的 Issue。

### 提交代码
1. Fork 本仓库
2. 从 `main` 分支创建你的开发分支：`git checkout -b feature/your-feature-name`
3. 编写代码并添加必要的测试
4. 提交代码：`git commit -m "feat: 简要描述你的改动"`
5. 推送到你的 Fork：`git push origin feature/your-feature-name`
6. 提交 Pull Request

### 提交规范
遵循 Conventional Commits 规范：
- `feat:` 新功能
- `fix:` 修复 Bug
- `docs:` 文档变更
- `refactor:` 代码重构
- `style:` 格式调整

### 代码审查
所有 PR 至少需要 1 位成员 Review 后方可合并。请确保：
- 代码可正常运行
- 新增功能有对应的测试或截图说明
- 代码风格与项目现有风格保持一致
